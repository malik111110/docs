---

The list endpoint gives you a snapshot of every verification record currently stored in the Quick KYC database. It is the primary data source for the audit dashboard and is useful for building compliance queues, generating summary reports, or checking the overall volume and status distribution of submissions. The response returns a flat array — there is no pagination at this time, so all records are returned in a single call.

## Endpoint

```
GET /api/verifications
```

## Request

No query parameters, headers, or request body are required.

```bash
curl http://your-server:8080/api/verifications
```

## Response — 200 OK

The response body is a JSON array. Each element is a verification summary object containing the fields described below.

<ResponseField name="id" type="string">
  Unique record identifier in `KYC-xxxxxxxx` format.
</ResponseField>

<ResponseField name="name" type="string">
  The subject's full name as extracted by OCR from the identity document. May be an empty string if the record is still `PENDING` and extraction has not completed.
</ResponseField>

<ResponseField name="email" type="string">
  The email address associated with the verification submission, if provided at the time of capture.
</ResponseField>

<ResponseField name="status" type="string">
  Current compliance status of the record. One of:
  - `PENDING` — queued or actively processing
  - `UNDER_REVIEW` — processing complete; flagged for manual review
  - `APPROVED` — cleared by the engine or a compliance officer
  - `REJECTED` — denied by the engine or a compliance officer
</ResponseField>

<ResponseField name="riskScore" type="integer">
  Composite fraud risk score from `0` to `100`. Higher values indicate greater suspicion. Scores above `75` typically trigger `UNDER_REVIEW` status automatically.
</ResponseField>

<ResponseField name="submittedAt" type="string">
  ISO 8601 timestamp of when the operator submitted the verification.
</ResponseField>
