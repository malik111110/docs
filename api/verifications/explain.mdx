The explain endpoint translates the raw risk signals and trust index numbers for a verification record into a single, human-readable paragraph. Rather than asking a compliance officer to interpret signal IDs like `r_shadow_overlay` or cross-reference confidence floats, you can surface this natural-language summary directly in your review interface so that decisions can be made quickly and with full context.

## Endpoint

```
GET /api/verifications/:id/explain
```

## Path Parameter

<ParamField path="id" type="string" required>
  The `KYC-xxxxxxxx` record ID of the verification you want an explanation for.
</ParamField>

### Example Request

```bash
curl http://your-server:8080/api/verifications/KYC-a9fd8241/explain
```

## Response — 200 OK

<ResponseField name="explanation" type="string">
  A plain-language paragraph generated by the local AI context engine, summarising the verification outcome. The explanation describes which status was assigned, which forensic signals were triggered, which extracted fields raised concerns, and what visual indicators contributed to the risk score.
</ResponseField>

### Example Response

```json
{
  "explanation": "This passport was flagged UNDER_REVIEW due to a 5-year expiration violation and localized shadow overlays that interfered with watermarking validation."
}
```

<Tip>
  Surface this endpoint's output directly in your compliance portal's case review panel. Compliance officers who are unfamiliar with technical signal identifiers like `r_expired` or `r_shadow_overlay` can read the plain-language summary and make an informed approve/reject decision without needing to interpret the raw `riskSignals` array. Pair it with the `PATCH /api/verifications/:id/status` call to complete the full review flow in a single screen.
</Tip>
